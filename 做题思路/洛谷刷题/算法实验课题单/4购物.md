# P1658 购物

## 题目描述

你就要去购物了，现在你手上有 $N$ 种不同面值的硬币，每种硬币有无限多个。为了方便购物，你希望带尽量少的硬币，但要能组合出 $1$ 到 $X$ 之间的任意值。

## 输入格式

第一行两个数 $X, N$，以下 $N$ 个数，表示每种硬币的面值。

## 输出格式

最少需要携带的硬币个数，如果无解输出 `-1`。

## 输入输出样例 #1

### 输入 #1

```
20 4
1 2 5 10
```

### 输出 #1

```
5
```

## 说明/提示

对于 $30\%$ 的数据，满足 $N \le 3$，$X \le 20$；

对于 $100\%$ 的数据，满足 $N \le 10$，$X \le 10^3$。

# 心得
注意贪心算法的正确性，没有反例的情况不能证明正确性也可能不对

## 最初思路
首先，最初要理解题目含义。要去购物，为了方便，这些词，再结合题意，意思是带的硬币可以凑出所有数额，但是不用全部凑一遍，只需要可以凑出即可。
在理解题意之后，可以从简单的例子入手，比如如果我们已经凑出了5元，那么手上再来一张5元，就可以凑出5-10元的范围。
到这里思路是对的，但是在实现时采用反向实现，即比如要凑9元，我们手上最大的是5元，那么就只需要凑max(4,5) = 5，既然0-5可以凑出来，那5-10也就可以凑出来了。
但是对于要凑10元的情况，两个5就可以凑出来，这又是特殊情况。
再经过多次边界调整后，仍然有几个算例不能通过，所以问题在于：我们的贪心策略，即优先选最大的，结合范围的定理选剩下的，是否有问题？
思考例子 1 4 5  凑出1-7，选择5后需要凑1-4，用到4个1，但选择4只需要凑1-3,4+3刚好又可以覆盖7，也就是选大的可以提供更大的范围，但是也会导致前面的范围变大，如果后面的两个数相差不大，但是前面的差的大，就会导致贪心解并非最优解。
## 正确思路
我们直接按照分析出来的思路正向求解，维护一个已得范围的右端点，加入范围能覆盖的最大硬币来扩展，这样能保证范围连续的情况下达到最小的数目。
核心部分代码如下：
```c++
while (sum < x) {
    int i = n;
    for (;i > 0; i--) {
        if (sum < coin[i]) break;
    }//找到最大可覆盖值
    ans++;
    sum += coin[i];
}
```
## 总结
对于贪心问题，一定要多举反例证明贪心解即最优解，也可以用交换论证来证明，总之要确定贪心解的正确性再下手才不会出错。
