#### 关于为什么修改时x + lowbit(x)的证明：
首先是举例：
10101  他的覆盖位置是：10110 11000 100000...剩下的是1左移
核心思想是：找原本比它大，去掉末尾1又比它小的数
严格证明：
1.需要满足：***i - lowbit(i) + 1 <= pos <= i***
右端点从pos开始，因为pos是满足 ***pos >= pos*** 的最小i,同时 ***pos - lowbit(pos) + 1 <= pos***
2.对于一个已经覆盖的i,我们知道 ***i - lowbit(i) + 1 <= pos***,也就是 ***i - lowbit(i) < pos***,
3.那么对于 ***j = i + lowbit(i),j > i >= pos***,右边满足
4.注意到 ***lowbit(j) >= 2 * lowbit(i)***，因为进位会更高，不进位也是2倍，那么 ***j − lowbit(j) + 1 ≤ i − lowbit(i) + 1 ≤ pos***说白了就是上面的核心思想
5.j一定是最紧的，因为如果是i + lowbit(i)还小的数，那么这个1在当前的后面，删掉他之后＋1的值要比pos大
